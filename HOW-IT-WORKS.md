# How This Site Works

A guide to understanding, running, and editing the Hypermobility MD website.

---

## Quick Start

```bash
npm install    # first time only
npm start      # launches dev server at http://localhost:8080
npm run build  # builds the site to _site/
```

---

## The Big Picture

This site uses three tools working together:

1. **Eleventy (11ty)** — a static site generator that turns templates + data into HTML pages
2. **Decap CMS** — a browser-based content editor at `/admin/` so your aunt can edit without code
3. **GitHub Pages + Actions** — free hosting that automatically rebuilds the site when changes are pushed

### How a new episode gets published

```
Aunt logs into yoursite.com/admin/
        ↓
Fills in episode form (title, guest, description, etc.)
        ↓
Clicks "Publish" → Decap CMS commits a new .md file to GitHub
        ↓
GitHub Actions detects the push → runs `npm run build`
        ↓
11ty reads all episode .md files → generates the site
        ↓
Built site deployed to GitHub Pages → live in ~30 seconds
```

## Key Concepts

### Templates (Nunjucks / .njk files)

Each page is a `.njk` file with two parts:

```
---
layout: base.njk          ← "wrap me in the base layout"
title: "About"             ← page title (used in <title> tag)
activePage: "about"        ← highlights the correct nav link
permalink: /about/         ← the URL this page will live at
---

<section class="hero">     ← the actual page content (plain HTML)
  ...
</section>
```

The **base layout** (`_includes/base.njk`) provides the `<html>`, `<head>`, navigation, footer, and theme switcher — so each page only contains its unique content.

### Episode Markdown Files

Each episode is a file like `src/episodes/184.md`:

```yaml
---
num: 184
title: "Why Doctors Miss EDS, POTS, and MCAS"
date: "2026-02-19"
duration: "1h 7m"
description: "In this episode..."
tags:
  - "EDS"
  - "POTS"
guests:
  - "Dr. Dacre Knight"
audioUrl: "https://..."
videoUrl: "https://youtu.be/..."
guestImages:
  - "Guests/Darce_Knight.jpg"
---

(transcript text goes here, below the ---)
```

The front matter (between `---`) is structured data. The body (after `---`) is the transcript.

### How data.js Gets Generated

The podcast pages use client-side JavaScript for search, filtering, and modals. This JS needs all episode data in a single `data.js` file.

Instead of maintaining `data.js` by hand, 11ty generates it automatically:

- `src/js/data.js.njk` is a Nunjucks template that loops over all episode markdown files
- At build time, 11ty reads every `src/episodes/*.md` file and injects the data
- The output is `_site/js/data.js` — a normal JavaScript file the browser can load

This means: **edit the markdown files, and data.js updates itself**.

### Decap CMS

Decap CMS is a React app that runs entirely in the browser. It:

1. Authenticates with GitHub (your aunt logs in with her GitHub account)
2. Reads the repo to show existing content
3. Provides form fields for editing (defined in `admin/config.yml`)
4. Commits changes directly to the GitHub repo when she clicks "Publish"

The `config.yml` defines one collection called "episodes" that maps to the `src/episodes/` folder. Each field in the config corresponds to a front matter field in the markdown files.

## Common Tasks

### Adding a new episode (via CMS)

1. Go to `yoursite.com/admin/`
2. Click "Podcast Episodes" → "New Podcast Episode"
3. Fill in the fields (episode number, title, date, guests, etc.)
4. Upload a guest photo if needed
5. Paste the transcript in the Transcript field
6. Click "Publish"

### Adding a new episode (manually)

1. Create a new file: `src/episodes/185.md`
2. Copy the front matter structure from an existing episode
3. Fill in the fields
4. Add the transcript below the `---`
5. If there's a new guest photo, drop it in `src/Guests/`
6. Commit and push to GitHub

### Adding a new guest photo

- Drop the image in `src/Guests/` (e.g., `Guests/New_Guest.jpg`)
- Add a mapping in `src/_data/guests.json` under `images`:
  ```json
  "new guest": "Guests/New_Guest.jpg"
  ```
- The key is the guest's name in lowercase, without "Dr." or credentials

### Editing site content (non-episode pages)

Edit the `.njk` files in `src/` directly. They're mostly plain HTML with a Nunjucks front matter header.

### Previewing changes locally

```bash
npm start
```

Opens at `http://localhost:8080` with live reload — changes appear instantly.

## Hosting Setup (GitHub Pages)

### One-time setup

1. Create a GitHub repo (e.g., `hypermobility-md`)
2. Edit `src/admin/config.yml` — change `repo:` to your actual `username/repo-name`
3. Push all code to the repo
4. Go to repo Settings → Pages → Source: select "GitHub Actions"
5. The deploy workflow runs automatically on every push to `main`

### Custom domain (optional)

1. In repo Settings → Pages → Custom domain, enter `hypermobilitymd.com`
2. Add a `CNAME` DNS record pointing to `your-username.github.io`
3. Create a file `src/CNAME` containing just `hypermobilitymd.com`
4. Add to `eleventy.config.js`: `eleventyConfig.addPassthroughCopy("src/CNAME")`

## Troubleshooting

**Build fails?** Run `npm run build` locally to see the error. Usually a YAML syntax issue in an episode file.

**CMS login not working?** Make sure the `repo` in `admin/config.yml` matches the actual GitHub repo, and that the GitHub account has write access.

**Episode not showing up?** Check that the markdown file has valid front matter (proper `---` delimiters, no YAML syntax errors).

**Guest photo not appearing?** The guest name in `guests.json` must be lowercase and match what's used in the episode's `guests` field (minus "Dr." and credentials).
